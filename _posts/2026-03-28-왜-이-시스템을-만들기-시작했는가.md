# 동기

- 게임업계는 IT업계 내에서도 AI 에이전트 활용도가 낮고, 본인도 뒤쳐져 있다는 자각
- 업계 전반 AX 진행 중 — 앞서나가고 싶었음
- Claude Code 작업 자체가 게임 플레이보다 재미있었음 → 본업(게임 개발)과 자연스럽게 연결

# 발전 경로

1. 발표자료 → PPT 자동화 (가벼운 실험)
2. 회사 기획팀용 기획서 제작 AI (실무 적용/검증)
3. 기획서 기반 구현 코딩 시스템 (기획→구현 파이프라인)
4. "내 것을 만들자" — 회사 프로젝트에만 쓰기엔 아까움

회사에서 실전 검증 → 능력치 축적 → 자기 시스템 구축 욕심

# 프로젝트 구조

- **GameMaker**: 1인~소규모 팀이 100인+ 팀 효율로 게임 개발하는 제작 툴. 판매 가능 수준이 목표.
- **GameMakerEngine**: 시스템 간 경쟁을 통해 GameMaker 모듈의 성능을 최적화하는 메타 시스템. "GameMaker를 찍어내는 프로젝트."
- **MissionDirector**: AI를 활용한 자기개발 프로젝트. AI 시대에 나란히/앞서서 달리기 위함.

# 핵심 의사결정: 규모

- 갈림길: 어느 정도 규모로 시작할 것인가
- GameMakerEngine — 거창하지만, 시스템 경쟁 모델에 대한 흥미가 지속성을 보장한다고 판단 → 속행
- MissionDirector — 두려움을 동력으로 전환. 뒤쳐질 바엔 직접 앞서나가자

# 현재 고민

게임 업계 내 향후 포지션 설계:
- 회사 소속 디렉터
- 게임 회사 AX 전문가
- 인디펜던트 AI 활용 게임개발자

미정. 단, 어느 길이든 현재 구축 중인 시스템들이 기반이 될 것이라는 확신은 있음.
